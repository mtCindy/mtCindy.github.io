发表于 2017-05-19

内边距不能是负值，但是外边距可以
外边距通常是透明的，从中可以看到父元素的背景
边框与内容和内边距有相同的颜色
边框的宽度绝对不能为负值

包含块：

  1. 每个元素都相对于其包含块摆放.
  2. 可以这么说，包含块就是一个元素的“布局上下文”
  3. 对于正常的西方语言文本流中的一个元素，包含块由最近的块级祖先框，表单元格或行内
    块祖先框的内容边界构成
  4. 行内元素的摆放方式并不直接依赖于包含块

术语：

  1. 正常流：
    这是指西方语言文本从左向右，从上向下显示，这也是我们熟悉的传统的HTML文档的文本布
    局。在非西方语言中，流方向可能不同。大多数元素都在正常的流中，要让一个元素不在正
    常流中，唯一的办法就是使之成为浮动或者定位元素。
  2. 非替换元素：
    如果元素的内容包含在文档中，则称之为非替换元素。
  3. 替换元素：
    这是指用作为其他内容占位符的一个元素。一个经典的例子就是img元素，它只是指向一个
    图像文件，这个文件将插入到文档流中该img元素本身所在位置。大多数表单元素也可以替
    换
  4. 块级元素：
    这是指段落，标题或div之类的元素，这些元素在正常流中时，会在其框之前和之后生成“换
    行”，所以处于正常流中的块级元素会垂直摆放。通过声明display：block，可以让元素生成
    块级框
  5. 行内元素：
    这是指strong或span之类的元素，这些元素不会在之前或者之后生成“行分隔符”，它们是块
    级元素的后代，通过声明display: inline可以让元素生成一个行内框
  6. 根元素：
    位于文档树顶端的元素，在HTML中，这就是元素html,在XML文档中，可以是该语言允许的任
    何元素

水平格式化:

水平格式化的“7大属性”是：margin-left, border-left, padding-left, width, padding-right, border-right,
margin-right,这些属性与块级框的水平布局有关系
这7个属性的值加在一起必须是元素包含块的宽度， 这往往是块元素的父元素的width值
（因为块级元素的父元素几乎都是块元素）
在这7个属性中，只有三个属性可以设置为auto: 元素内容width, 以及左右外边距，其余属
性必须设置为特定的值，或者默认宽度为0

使用auto（从某种程度上讲，可以用auto弥补实际值与所需总和的差距）

  1. 如果设置width，margin-left或margin-right中的某个值为auto，而余下两个属性指定为
    特定的值，那么设置为auto的属性会确定所需要的长度，从而使元素框的宽度等于父元素的width,
    举个例子：
    假设7个属性的值必须为400px, 没有设置内边距或边框，右外边距是和width设置为100px,
    左外边距设置为auto, 那么左外边距的宽度是200px
  2. 如果这三个值都设置为auto
    此时会把margin-right强制设置为auto
  3. 如果两个外边距都显示地设置，而width设置为auto
    width值将设置为某个所需要的值
  4. 不止一个auto
    如果两个外边距都设置为auto， 它们会设置为相等的长度，因此将元素在其父元素中居中
  5. 将某个外边距以及width设置为auto，设置为auto的外边距会减为0
  6. 3个属性都设置为auto, 两个外边距都会设置为0，width尽可能宽
    注意：由于水平外边距不会合并，父元素的外边距，内边距和边框可能会影响子元素，这种效果是
    间接的

负外边距


如果左外边距为负，不仅段落会超出div的边框，还会超出浏览器本身
