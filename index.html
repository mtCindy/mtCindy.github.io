关于重构的几点想法

发表于 2017-05-19

为什么要重构？

重构是什么？

重构，通常指的是【代码重构】

对软件内部结构的一种调整，目的是在不改变软件可观察行为的前提下，提高其可理解性，
降低其修改成本

重构本质上是一种代码整理技术，这项技术使得代码整理的效率更高，风险更小

如何做？

几个出发点：

  1. 什么时候开始
  2. 什么时候停止
  3. 前提条件
  4. 重构的过程
    测试保护
    识别味道
    采用手法
    运行测试
    提交代码

重构与设计的关系？

有了重构技术，我们不用在前期做非常详细的设计，做适当的设计，然后通过重构让设计浮
现出来。不用在乎软件是否符合模式，只要符合原则即可

css盒子模型由了解到深入

发表于 2017-05-19

内边距不能是负值，但是外边距可以
外边距通常是透明的，从中可以看到父元素的背景
边框与内容和内边距有相同的颜色
边框的宽度绝对不能为负值

包含块：

每个元素都相对于其包含块摆放;可以这么说，包含块就是一个元素的“布局上下文”
对于正常的西方语言文本流中的一个元素，包含块由最近的块级祖先框，表单元格或行内块祖
先框的内容边界构成
行内元素的摆放方式并不直接依赖于包含块

术语：

  1. 正常流：
    这是指西方语言文本从左向右，从上向下显示，这也是我们熟悉的传统的HTML文档的文本布局。
    在非西方语言中，流方向可能不同。大多数元素都在正常的流中，要让一个元素不在正常流中，
    唯一的办法就是使之成为浮动或者定位元素。
  2. 非替换元素：
    如果元素的内容包含在文档中，则称之为非替换元素。
  3. 替换元素：
    这是指用作为其他内容占位符的一个元素。一个经典的例子就是img元素，它只是指向一个图像文件，
    这个文件将插入到文档流中该img元素本身所在位置。大多数表单元素也可以替换
  4. 块级元素：
    这是指段落，标题或div之类的元素，这些元素在正常流中时，会在其框之前和之后生成“换行”，
    所以处于正常流中的块级元素会垂直摆放。通过声明display：block，可以让元素生成块级框
  5. 行内元素：
    这是指strong或span之类的元素，这些元素不会在之前或者之后生成“行分隔符”，它们是块级元素的
    后代，通过声明display: inline可以让元素生成一个行内框
  6. 根元素：
    位于文档树顶端的元素，在HTML中，这就是元素html,在XML文档中，可以是该语言允许的任何元素

水平格式化:

水平格式化的“7大属性”是：margin-left, border-left, padding-left, width, padding-right, border-right,
margin-right,这些属性与块级框的水平布局有关系
这7个属性的值加在一起必须是元素包含块的宽度， 这往往是块元素的父元素的width值（因为块级元素的父元素
几乎都是块元素）
在这7个属性中，只有三个属性可以设置为auto: 元素内容width, 以及左右外边距，其余属性必须设置为
特定的值，或者默认宽度为0

使用auto（从某种程度上讲，可以用auto弥补实际值与所需总和的差距）

  1. 如果设置width，margin-left或margin-right中的某个值为auto，而余下两个属性指定为
    特定的值，那么设置为auto的属性会确定所需要的长度，从而使元素框的宽度等于父元素的width,
    举个例子：
    假设7个属性的值必须为400px, 没有设置内边距或边框，右外边距是和width设置为100px,
    左外边距设置为auto, 那么左外边距的宽度是200px
  2. 如果这三个值都设置为auto
    此时会把margin-right强制设置为auto
  3. 如果两个外边距都显示地设置，而width设置为auto
    width值将设置为某个所需要的值
  4. 不止一个auto
    如果两个外边距都设置为auto， 它们会设置为相等的长度，因此将元素在其父元素中居中
  5. 将某个外边距以及width设置为auto，设置为auto的外边距会减为0
  6. 3个属性都设置为auto, 两个外边距都会设置为0，width尽可能宽
    注意：由于水平外边距不会合并，父元素的外边距，内边距和边框可能会影响子元素，这种效果是
    间接的

负外边距


如果左外边距为负，不仅段落会超出div的边框，还会超出浏览器本身

前端代码规范

  1. 尽量避免用var,多用let和const
  2. 定义对象用对象字面量

  3. 定义对象的属性和方法，尽量用速写法

  5. 用… to copy arrays

  7. 尽量用解构赋值符

  8. 定义string尽量用’’

  11. switch…case…

  12. 取值判断

async, await, Promise之间的关系

发表于 2017-05-19

Promise

Promise对象用于异步计算，一个Promise表示一个现在或将来可用，亦或永远不可用的值

语法

描述

Promise 对象是一个代理对象（代理一个值），被代理的值在Promise对象创建时可能是未知的。它允许你为异步代码执行结果的成功和失败分别绑定相应的处理方法（handlers ）。 这让异步方法可以像同步方法那样返回值，但是并非立即返回执行的结果，因为毕竟执行的是异步代码，因此，它会返回一个Promise对象，如前所说，它是一个代理的对象，代理了最终返回的值，可以在后期使用

一个 Promise有以下几种状态:

pending: 初始状态，未履行或拒绝。
fulfilled: 意味着操作成功完成。
rejected: 意味着操作失败。
pending 状态的 Promise 对象可能以 fulfilled 状态返回了一个值，也可能被某种理由（异常信息）拒绝（reject）了。当其中任一种情况出现时，Promise 对象的 then 方法绑定的处理方法（handlers ）就会被调用（then方法包含两个参数：onfulfilled 和 onrejected，它们都是 Function 类型。当值被填充时，调用 then 的 onfulfilled 方法，当Promise被拒绝时，调用 then 的 onrejected 方法， 所以在异步操作的完成和绑定处理方法之间不存在竞争）。
因为 Promise.prototype.then 和 Promise.prototype.catch 方法返回 promise 对象自身， 所以它们可以被链式调用。

async function

async function函数定义了一个异步函数，它返回一个AsyncFunction对象

描述：

调用异步函数时会返回一个 promise 对象。当这个异步函数返回一个值时，promise 的 resolve 方法将会处理这个返回值；当异步函数抛出的是异常或者非法值时，promise 的 reject 方法将处理这个异常值。
异步函数可能会包括await 表达式, 这将会使异步函数暂停执行并等待promise解析传值后, 继续执行异步函数并返回解析值。

举例：

await

await 操作符用于等待一个async function返回的一个Promise

语法

[return_value] = await expression;

返回值

返回promise的解析值，如果不是promise, 则返回值本身

表达式

一个promise或者任何等待解析的值

描述：

await 表达式会造成异步函数停止执行并且等待promise的解决，当值被resolved，异步函数会恢复执行以及返回resolved值。如果该值不是一个promise，它将会被转换成一个resolved后的promise。
如果promise被rejected，await表达式会抛出异常值。

举例：

如果一个Promise被传递到一个await表达，它会等待Promise的解析并返回解析的值

如果该值不是一个Promise，它将该值转换为已解决的Promise，并等待它

如果Promise被拒绝，则拒绝的值被抛出
